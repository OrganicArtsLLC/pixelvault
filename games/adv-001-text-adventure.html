ADV-001 · TEXT ADVENTURE

ADV — Adventure Games Prototype #001 ★★★★☆

How to Play

Type commands to explore a world made entirely of words. Navigate rooms, collect items, solve puzzles, and gather five treasures to win.

  • Type a command and press Enter to act
  • R at game over — Restart
  • ? — Toggle this info panel

Commands

  • Movement: go north, south, east, west, up, down (or just n, s, e, w, u, d)
  • Look: look — redisplay the current room
  • Take/Get: take lamp — pick up an item
  • Drop: drop lamp — leave an item in the room
  • Inventory: inventory (or i) — see what you carry
  • Examine: examine sword — look closely at an item
  • Use: use key, use sword — use an item contextually
  • Open: open door, open mailbox — interact with objects
  • Help: help — list available commands

History & Lineage

Ancestry: Colossal Cave Adventure (1976) → Zork (1977) → Infocom (1979-1989) → modern interactive fiction

Zork began as a mainframe game at MIT, written by Tim Anderson, Marc Blank, Bruce Daniels, and Dave Lebling on a PDP-10. It was the successor to Will Crowther and Don Woods' Colossal Cave Adventure, the first text adventure.

What made Zork special was its parser. While Adventure understood two-word commands, Zork could parse complex sentences: "put the jewel-encrusted egg in the trophy case." This was natural language processing in 1977 — years before the term "NLP" entered common use.

Infocom commercialized Zork and built a dynasty of text adventures, each pushing the parser and world model further. The company's philosophy: the best graphics are in the player's imagination.

AI Archaeology

AI Era: 2 — Expert Systems (1965-1979)

Contemporary AI: Zork's parser was a rule-based expert system. It tokenized input, identified verbs and nouns, resolved ambiguity ("which door?"), and mapped commands to world-state changes. This is the same architecture as contemporary expert systems in medicine and engineering.

Hidden Intelligence: The game world is a knowledge graph — rooms are nodes, exits are edges, items have properties and affordances. The parser performs symbolic reasoning: "unlock door" requires checking (has key?) AND (door locked?) AND (in correct room?). This is predicate logic applied to interactive narrative.

The Convergence: Modern LLMs can now play Zork-style games by generating commands from room descriptions. The wheel has turned — AI went from being the game's parser to being the game's player. The text adventure is the original Turing test for language understanding.

If AI Had Designed This: An LLM-designed text adventure would have infinite rooms generated on the fly, but would lose the hand-crafted puzzle design that makes Zork satisfying. The constraint of 12 rooms forces elegant puzzle interconnection.

Classification

Series: ADV — Adventure Games

Games where exploration and puzzle-solving through a world model are the core mechanics.

Core Mechanic

Text parser navigation with inventory puzzles. Type natural language commands. The parser interprets intent. The world responds. No graphics — pure language as interface.

What This Prototype Tests

  • Parser feel — does typing commands feel powerful or frustrating?
  • Room descriptions — do words alone create a sense of place?
  • Puzzle logic — do item-based puzzles feel fair and discoverable?
  • Terminal aesthetic — does the green-on-black canvas evoke the era?

Prototype Notes

This is a sketch rated 4/5. A 12-room text adventure with a two-word parser, inventory system, and 5-treasure win condition. All rendering is canvas-based monospace text — no DOM elements for gameplay. The classic terminal feel is deliberate homage.
