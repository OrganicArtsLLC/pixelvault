PLT-002 · DIG AND RUN

PLT — Platformer Prototype #002 ★★★★☆

How to Play

Collect all gold pieces, then reach the top to escape. Dig holes to trap enemies and create paths. You cannot jump — only move, climb ladders, and dig.

  • ← → or A D — Move left/right
  • ↑ or W — Climb up ladder
  • ↓ or S — Descend / fall through
  • Z — Dig left
  • X — Dig right
  • R — Restart
  • ? — Toggle this info panel

Tips

  • No jump: You fall whenever there's no floor, ladder, or rope beneath you.
  • Digging: Creates temporary holes. Enemies fall in and are trapped until the hole fills.
  • Ropes: You can traverse along ropes (horizontal bars) — hang and move.
  • Trapped enemies: They regenerate after a few seconds. Don't be nearby when they do.

History & Lineage

Ancestry: Lode Runner (1983, Doug Smith/Broderbund) → Championship Lode Runner (1984) → Spelunky (2008, inspired by)

Lode Runner (1983) by Doug Smith, a college student, was one of the first games with a level editor — spawning a user-generated content community decades before Minecraft. The core insight: removing jump from a platformer turns it into a puzzle game. Without jump, the player must think about path planning, dig strategically, and use enemies' behavior patterns.

The dig mechanic serves dual purposes: create paths through solid ground, AND trap enemies. This dual-use verb is elegant design — one action, two strategic applications.

AI Archaeology

AI Era: 3 — Knowledge Explosion (1978–1983)

Contemporary AI: Pathfinding algorithms (A*) were well-established. Planning with environment modification was not.

Hidden Intelligence: Lode Runner is a planning problem where the action space includes environment modification. The player must reason about dig-fill cycles, enemy pathing, and gold collection order. Enemies that regenerate from traps create temporal constraints — the player must plan sequences that account for time. This is a form of temporal planning with destructive and constructive actions, a class of problem that PDDL (Planning Domain Definition Language) wouldn't formalize until 1998.

The Convergence: Modern game-playing agents struggle with Lode Runner because the solution requires environment modification planning — something that standard RL doesn't handle well. The game requires a world model.

If AI Had Designed This: Procedurally generated levels with guaranteed solvability — a constraint satisfaction problem over dig sequences and enemy patrol routes.

Classification

Series: PLT — Platformer

Lode Runner removes the defining platformer verb (jump) and replaces it with dig. This transforms the genre from reflexive to strategic.

Core Mechanic

Dig traps, collect gold, no jump. Ladder climbing. Rope traversal. Temporary holes that trap and regenerate. Gold collection + escape objective.

What This Prototype Tests

  • No-jump constraint — does removing jump make movement feel puzzle-like?
  • Dig as dual-use verb — does digging feel useful for both paths and traps?
  • Enemy trapping — does the trap-and-escape cycle feel satisfying?
  • Level as puzzle — does gold collection order matter?
