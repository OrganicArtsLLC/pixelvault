SPT-002 · MULTI ATHLETICS

SPT — Sports Prototype #002 ★★★★☆

How to Play

Compete in four athletic events. Alternate-mash Left and Right arrow keys to build speed. Use Space for timed actions (jump, throw, clear).

  • ← → — Alternate mash to build running speed
  • Space — Jump / throw / action (event-specific)
  • Enter — Start event / proceed to next
  • R — Restart current event or full game
  • ? — Toggle this info panel

Events

  • 100m Dash: Mash L/R as fast as possible. Pure speed race against the clock.
  • Long Jump: Mash to build speed on the runway. Press Space at the foul line to jump. Timing determines angle — optimal is 45°.
  • Javelin: Mash to build speed. Press and hold Space — an angle meter oscillates. Release to throw at the selected angle.
  • High Jump: Press L/R to approach the bar. Press Space to jump. Press Space again at peak height to arch over the bar.

Tips

  • Alternate cleanly: Pressing the same key twice in a row does nothing — you must alternate L and R.
  • Watch the speed meter: Speed decays if you stop mashing.
  • Long Jump timing: Too early = short. Too late = foul. Aim for just before the line.
  • Javelin angle: Around 40–45° gives maximum distance at high speed.

History & Lineage

Ancestry: Track & Field (1983, Konami) → Summer Games (1984, Epyx) → Decathlon (1984, Activision)

Konami's Track & Field (1983) introduced button-mashing as a primary game mechanic in arcades. Players wore out joysticks and spoons became the instrument of choice for rapid alternation. Epyx expanded the concept with Summer Games (1984), adding multiple event types with varied control schemes — the same inputs meant different things in each event.

The multi-event format forced players to rapidly switch mental models: pure speed optimization in sprints, timing optimization in jumps, angle selection in throws. This cognitive flexibility requirement made these games surprisingly deep despite simple inputs.

AI Archaeology

AI Era: 3 — Knowledge Explosion (1978–1983)

Contemporary AI: Multi-task learning was decades away. Transfer learning across domains was theoretical.

Hidden Intelligence: Each event is a different optimization problem sharing the same interface. The 100m is pure throughput maximization. Long jump adds a timing constraint. Javelin introduces continuous angle selection. High jump requires sequential timing decisions. The player must rapidly switch between these optimization strategies using identical inputs — this is transfer learning across related domains, performed by human neural networks in real-time.

The Convergence: Modern multi-task RL agents face this exact challenge: shared input/output spaces across tasks with different reward functions. The button-mashing interface is a bandwidth-limited channel — the player's maximum input rate IS the constraint, making it a physical optimization problem.

If AI Had Designed This: Optimal mashing frequency analysis, biomechanically-informed timing windows, and adaptive difficulty based on measured human input bandwidth.
